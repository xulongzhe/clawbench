## Description
When the context is cancelled during the scan loop in `CLIBackend.ExecuteStream` (cli_backend.go:132-144) or `CodexBackend.ExecuteStream` (codex_stream.go:543-554), the goroutine returns immediately without calling `cmd.Wait()`. This leaves the child process in a "waitable" state — the OS process descriptor is not released, causing a goroutine leak (the Go runtime keeps a reference for Wait4). Over time with many cancelled sessions, this leaks file descriptors and goroutines.

## Impact
Goroutine and file descriptor leak on every cancelled AI session. Under heavy use, this could exhaust system resources.

## Suggestion
Before returning on context cancellation, call `cmd.Process.Kill()` followed by `cmd.Wait()` to properly clean up the process:
```go
cmd.Process.Kill()
cmd.Wait()
```
