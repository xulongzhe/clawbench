## Description
Ring buffer data corruption — `readPTY` reuses the same `buf` slice for each read. After writing data into the ring buffer, the next read overwrites the same underlying array. Since the ring buffer stores slice references (not copies), stored entries point to the same memory and are silently corrupted when the buffer is reused.

## Impact
Terminal replay data is corrupted: users see garbage characters or the wrong output when replaying terminal sessions. Historical terminal data is permanently lost.

## Suggestion
Copy the data from `buf` into a new slice before storing it in the ring buffer. Use `append([]byte{}, buf[:n]...)` or `make([]byte, n)` + `copy()` to ensure each ring entry owns its data.
