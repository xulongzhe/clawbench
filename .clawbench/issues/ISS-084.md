## Description
Qoder stream args builder doesn't use `buildBaseStreamArgs` — it constructs CLI arguments independently, diverging from other backends that use the shared builder. This means features added to the base builder (e.g., new flags, environment variables) are not applied to Qoder.

## Impact
Qoder sessions may miss features, flags, or environment variables that are automatically applied to other backends. Bug fixes in the shared builder don't propagate to Qoder, causing inconsistent behavior.

## Suggestion
Refactor Qoder to use `buildBaseStreamArgs` for constructing its CLI arguments, adding Qoder-specific flags on top of the base args. If Qoder needs fundamentally different arg structure, extract the common parts into a shared helper.
