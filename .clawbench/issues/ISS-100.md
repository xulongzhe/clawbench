## Description

When `TrySetSessionRunning` successfully marks a session as running (session_runtime.go:75-84), no WebSocket event is emitted to notify clients of the state transition. The calling code assumes the event was published, but clients never receive the "running" state.

## Impact

WebSocket clients never receive "running" state when a new AI execution starts. The UI misses the running indicator for the first message, making it appear as though nothing is happening despite an active AI session.

## Suggestion

Emit a "running" event inside `TrySetSessionRunning` when it successfully marks a session as running. This ensures all state transitions are communicated to connected clients.
