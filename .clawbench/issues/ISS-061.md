## Description
CLIBackend does not kill the CLI process group on context cancellation — only the lead process is killed. Child processes spawned by the CLI (e.g., shell subshells, build tools) become orphans and continue running.

## Impact
Orphaned child processes consume system resources (CPU, memory, file descriptors) indefinitely. On resource-constrained systems, this can lead to exhaustion and denial of service.

## Suggestion
Use process groups: set `Cmd.SysProcAttr.Setpgid = true` and kill the entire process group with `syscall.Kill(-pid, syscall.SIGKILL)` on cancellation. Alternatively, use `exec.CommandContext` with a custom cancel function that sends the signal to the process group.
