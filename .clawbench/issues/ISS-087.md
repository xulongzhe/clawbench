## Description
Race between Close() and waitProcess() — both can attempt to close the `ptmx` file descriptor. If Close() is called while waitProcess() is still running, the double close causes a panic or closes an unrelated file descriptor that was reused.

## Impact
Terminal session crash on close. In the worst case, a file descriptor reused by another goroutine is closed, causing data loss or corruption in an unrelated subsystem.

## Suggestion
Use `sync.Once` or an atomic flag to ensure `ptmx.Close()` is called exactly once. Have both Close() and waitProcess() go through the same guarded close path. Check the flag before closing and skip if already closed.
