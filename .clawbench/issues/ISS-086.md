## Description
ForceCancelSession does not clean up `activeSessions` or `sessionStreams` — after force-cancelling a session, the runtime metadata is left in memory. Subsequent lookups return stale session entries pointing to closed channels and nil streams.

## Impact
Resource leak: session entries accumulate in memory and are never garbage collected. Stale entries can cause nil pointer dereferences or reads from closed channels when accessed. Over time, this degrades server performance.

## Suggestion
In ForceCancelSession, remove the session from `activeSessions` and `sessionStreams` after cancellation. Ensure cleanup happens regardless of cancellation success. Add a defensive nil/stale check on session lookup.
