## Description
Empty AllowedPorts defaults to "allow all" including privileged ports (<1024). If `ssh.allowed_ports` is not configured, the validation logic treats an empty list as "no restrictions", allowing tunnels to any port including privileged service ports.

## Impact
An authenticated SSH user can tunnel to any internal service, including those on privileged ports (e.g., databases on 5432, Redis on 6379). This expands the attack surface beyond the intended scope.

## Suggestion
Change the default behavior: an empty AllowedPorts should deny all tunnels, not allow all. Require explicit configuration of allowed ports. Alternatively, set a safe default list (e.g., common HTTP ports) and block privileged ports (<1024) unless explicitly listed.
