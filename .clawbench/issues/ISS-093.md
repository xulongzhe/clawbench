## Description
RecoverFromCorruption ignores `db.Close()` error — after attempting database recovery, the close operation's error is silently discarded. If the close fails, it may indicate unwritten WAL data or a corrupted page that was not flushed, meaning the "recovered" database is actually in a bad state.

## Impact
Database connection leak if close fails and the connection is not properly released. Worse, the corruption may not actually be recovered, but the code proceeds as if it was, leading to data loss on next open.

## Suggestion
Handle the close error: log it at minimum, and if it indicates data loss, return an error to the caller. Consider retrying the close or using `db.Close()` in a deferred cleanup with error propagation.
