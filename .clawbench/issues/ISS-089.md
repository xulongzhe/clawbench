## Description
queue_consume SSE event field mismatch — the frontend (`useChatRender.ts`) expects a `files` field while the backend (`chat.go`) sends `filePaths`. This field name mismatch means the queue consume event's file attachments are never processed by the frontend.

## Impact
File attachments referenced in queue_consume events are silently dropped. Users do not see file references that the AI intended to surface, breaking the expected workflow.

## Suggestion
Align the field name between frontend and backend. Either rename the backend field to `files` or update the frontend to read `filePaths`. Add a TypeScript interface for SSE events to catch such mismatches at compile time.
