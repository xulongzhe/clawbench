## Description
`convertAudioLinks` (chatRenderUtils.ts:46) constructs `<audio src>` HTML attributes via string interpolation without escaping special characters. URLs containing `"`, `>`, or other HTML metacharacters could break out of the `src` attribute and inject arbitrary HTML.

## Impact
An attacker who can control the content of chat messages (e.g., through a malicious AI response or by manipulating SSE events) could inject arbitrary HTML or JavaScript via crafted audio URLs. This is an XSS vulnerability.

## Suggestion
Use proper HTML attribute escaping when constructing the `<audio src>` attribute. Replace `"` with `&quot;`, `<` with `&lt;`, `>` with `&gt;`, and `&` with `&amp;` in the URL before inserting it into the HTML string. Alternatively, use DOM API (`document.createElement`) to construct the element safely.
