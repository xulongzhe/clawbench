## Description
No connection limit on SSH tunnel server — each incoming connection spawns a goroutine and holds file descriptors. Without a cap, an attacker can open many connections to exhaust goroutines and file descriptors.

## Impact
Denial of service via resource exhaustion. The server becomes unresponsive when it runs out of file descriptors or goroutine stack space.

## Suggestion
Add a configurable maximum connection limit (e.g., `ssh.max_connections` in config). Use a semaphore (`sync.WaitGroup` with limit, or `chan struct{}`) to gate new connections. Reject connections over the limit with a clean error message.
