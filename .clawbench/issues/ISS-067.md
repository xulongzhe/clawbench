## Description
TOCTOU race on HasRunningExecutions + TriggerTask — the check for a running execution and the trigger of a new one are not atomic. Two concurrent trigger requests can both pass the "no running execution" check and start duplicate runs.

## Impact
Duplicate task executions running concurrently, potentially causing double side effects (e.g., duplicate commits, duplicate API calls, resource contention).

## Suggestion
Use a database-level transaction or advisory lock to make the check-and-insert atomic. Alternatively, use an atomic compare-and-swap on an in-memory flag per task ID. The flag should be set before spawning the goroutine and cleared only when the execution completes.
