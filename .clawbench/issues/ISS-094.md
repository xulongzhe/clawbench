## Description
InsertChunks race condition on maxID — multiple concurrent InsertChunks calls read and update the max chunk ID without proper synchronization. Two concurrent inserts can read the same maxID, assign overlapping IDs, and one will overwrite the other's data.

## Impact
RAG index corruption: chunks with duplicate IDs cause data loss where one insert silently overwrites another. Search results may be incomplete or incorrect.

## Suggestion
Use a database transaction or `INSERT` with auto-increment to let SQLite handle ID assignment atomically. Alternatively, protect the maxID read-and-increment with a mutex or use `sql.Result.LastInsertId()` after each insert.
