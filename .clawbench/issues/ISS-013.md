## Description
`UpdateTaskStats` and success-path UPDATE can result in stale task snapshot overwriting status changes made during execution. If the user paused the task while it was running, the completion handler would overwrite "paused" back to "active" via `newStatus = task.Status` (stale snapshot).

## Impact
Task status can be incorrectly reverted. A paused task may be un-paused when an execution completes.

## Suggestion
Read the current DB status before writing it back, or only update `last_run_at` and `run_count` in `UpdateTaskStats` without touching the `status` column for cancelled/failed executions.
