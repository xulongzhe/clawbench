## Description
useCrudList singleton key collision — the composable uses a shared module-level `Map<string, State>` keyed by `apiPrefix`. Different features using the same `apiPrefix` (e.g., terminal shortcuts and chat quick-send) share all reactive state: items, loading flags, error messages, etc.

## Impact
Cross-feature state contamination: CRUD operations in one feature (e.g., loading items) affect the UI state of an unrelated feature sharing the same prefix. Loading spinners appear in wrong components, items are replaced, errors are displayed incorrectly.

## Suggestion
Include a namespace or feature identifier in the map key, not just `apiPrefix`. For example, use `${featureId}:${apiPrefix}` as the key. Alternatively, do not use module-level singletons — return fresh state per composable call.
