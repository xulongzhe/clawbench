## Description
The deleteExecution action in the handler checks task ownership but never verifies that the execution belongs to that task. A user who owns task A can delete an execution from task B by supplying task B's execution ID. The DeleteTaskExecution function looks up the execution by its own ID without cross-checking the task_id.

## Impact
Authorization bypass — cross-task execution deletion. Any authenticated user can delete any task execution in the system, including from projects they don't own (if they know the execution ID).

## Suggestion
In DeleteTaskExecution, add a WHERE task_id = ? filter to the SELECT and DELETE, or verify in the handler that the execution's task_id matches the taskID from the URL path.
