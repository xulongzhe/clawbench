## Description
Cron-triggered auto executions can overlap for the same task with no guard. The manual trigger path checks `HasRunningExecutions` and rejects if already running, but the cron callback has no such guard.

## Impact
If a task's AI execution takes longer than the cron interval, the cron will fire again and start a second concurrent execution. This leads to resource contention, duplicate work, and confusing UI state where a single task has multiple simultaneous "running" executions.

## Suggestion
Add `HasRunningExecutions` check inside the cron callback before calling `executeTask`, same as the handler does for manual triggers.
