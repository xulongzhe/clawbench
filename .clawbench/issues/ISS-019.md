## Description
`ListFiles` walks `projectPath` from cookie without validating it is under `WatchDir`. A tampered `clawbench_project` cookie could point to an arbitrary directory (e.g. `/etc`), and `filepath.Walk` would recursively enumerate every file. All other file handlers use the same `requireProject` path without WatchDir boundary check.

## Impact
Full filesystem enumeration and file content access if cookie is tampered. Path traversal vulnerability.

## Suggestion
Add `isPathUnderBase(projectPath, watchAbs)` validation inside `requireProject` or at the top of every file handler.
