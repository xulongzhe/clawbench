## Description
There is a race window between `TrySetSessionRunning` (chat.go:328-364), goroutine spawn, and `RegisterSessionStream`. During this window, an SSE client connecting for the session will find no stream registered and cannot receive events.

## Impact
SSE clients connecting during this window will fail to subscribe to the session stream, missing all events from the beginning of the session. This manifests as an empty chat view that never updates.

## Suggestion
Register the session stream before spawning the goroutine, or use a two-phase approach where `TrySetSessionRunning` also pre-registers an empty stream buffer that the goroutine later populates. The stream registration should happen atomically with the session state change.
