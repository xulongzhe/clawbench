## Description
Codex resume parser ERROR detection returns without emitting a `done` event. When an ERROR line is detected during the resume scan, the parser exits early but does not send the terminal `done` event to the stream consumer.

## Impact
The SSE stream hangs indefinitely — the client never receives the `done` event and keeps the connection open, waiting for more data. The session appears "stuck" from the user's perspective.

## Suggestion
Emit a `done` event (with appropriate error info) before returning from the ERROR detection path. Ensure all exit paths in the parser emit `done` so the stream consumer can finalize the session.
