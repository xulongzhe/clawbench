## Description
In Codex resume mode, stdoutPipe is created (line 489) but never consumed (lines 506-513 only read stderrPipe). If Codex writes to stdout, the buffer fills up and the process blocks (deadlock). The stdout io.ReadCloser is never closed, leaking a file descriptor.

## Impact
FD leak on every Codex resume session. Potential deadlock if Codex writes significant output to stdout, causing the session to hang indefinitely.

## Suggestion
Move stdoutPipe creation into the non-resume branch. Set cmd.Stdout = io.Discard for resume mode before cmd.Start(), or drain stdout in a separate goroutine.
