## Description
`DiscoverClaudeModels` uses hardcoded "claude" via `exec.LookPath`, ignoring the agent's custom `Command` field. If a user specifies `command: /usr/local/bin/claude-custom` in their agent YAML, model discovery still runs `strings` on whichever `claude` is first on PATH.

## Impact
Models discovered may not match the binary actually invoked at chat time, leading to "model not found" errors or missing models for custom CLI configurations.

## Suggestion
Pass the agent's `Command` into `DiscoverModels`/`DiscoverClaudeModels` so discovery runs against the actual binary the user configured.
