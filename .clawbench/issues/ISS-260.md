## Description
`serveGitDeleteWorktree` (git.go:1744) passes `deletePath` as a git command argument without a `--` separator. A worktree path starting with dashes (e.g., `--force`) could be interpreted as git flags rather than a path argument.

## Impact
Command injection via flag injection. An attacker who can control the worktree path (e.g., through a crafted API request) could inject git flags, potentially causing unintended git operations or data loss.

## Suggestion
Add `--` before the path argument in the git command to signal end of options. For example: `git worktree remove -- <path>`. This ensures the path is never interpreted as a flag, even if it starts with dashes.
