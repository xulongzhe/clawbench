## Description
`executeStreamRun` event loop has no `case <-ctx.Done()` branch. The `for { select { case event, ok := <-eventCh: ... case <-flushTicker.C: ... } }` loop only exits on channel closure or a "done" event. If the CLI process hangs or a backend fails to close its event channel on context cancellation, this goroutine blocks indefinitely.

## Impact
Potential goroutine leak when CLI hangs silently or when backend implementations don't properly close event channels on cancellation. Can accumulate leaked goroutines over time, leading to memory pressure.

## Suggestion
Add `case <-ctx.Done():` to the select in the event loop, triggering `finalizeStreamRun` on cancellation regardless of event flow.
