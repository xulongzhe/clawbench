## Description
`chatSummaryEnabled` (session_runtime.go:319) is read by `triggerChatSummarization()` from the session completion goroutine and written by `SetChatSummaryEnabled()` (session_runtime.go:330) from HTTP handler goroutines without any synchronization. This is a data race on a non-atomic bool variable.

## Impact
Undefined behavior per the Go spec. The race detector will flag this. In practice, the read may see a stale value (either always-true or always-false depending on CPU cache), but the behavior is formally undefined.

## Suggestion
Use `atomic.Bool` (Go 1.19+) or protect with `activeMu`. Replace `var chatSummaryEnabled = true` with `var chatSummaryEnabled atomic.Bool` initialized to `true`, then use `.Load()` / `.Store()` everywhere.
