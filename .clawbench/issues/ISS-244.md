## Description
`JSON.parse(e.data)` is called without try/catch at multiple locations (useChatStream.ts:292,308,325,385,403,473,510,538). Any malformed SSE event data will throw an exception that crashes the entire SSE event listener, permanently breaking the chat stream.

## Impact
A single malformed event from the server (e.g., due to a bug in JSON serialization, network corruption, or encoding mismatch) terminates the SSE listener. The user sees the chat stop updating and must manually refresh the page.

## Suggestion
Wrap all `JSON.parse(e.data)` calls in try/catch blocks. On parse failure, log the error and the raw event data (truncated) for debugging, then continue processing subsequent events. Consider adding a counter for parse failures and triggering a reconnect if the failure rate exceeds a threshold.
