## Description

User-supplied SHA hashes and branch names are passed directly to git commands without validation at git.go:337, 552, 561, 268, and 1029. A crafted SHA like `--exec-path=/tmp/evil` would be interpreted as a git flag rather than a commit identifier.

## Impact

Could lead to arbitrary command execution or information disclosure through git flag injection. An attacker can craft request parameters that are interpreted as git command-line flags, potentially executing arbitrary programs or leaking system information.

## Suggestion

Validate SHAs with regex `^[0-9a-f]{4,40}$` before passing to any git command. Use `--` separator for branch names in checkout operations to prevent flag interpretation. Apply input sanitization consistently across all git command construction sites.
