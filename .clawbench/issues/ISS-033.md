## Description
`http.Request` is captured by a goroutine that outlives the HTTP handler. The goroutine uses `r` for i18n lookups via `T(r, ...)` calls inside `executeStreamRun` and the drain loop. After the handler returns, Go's HTTP server may recycle the request object via `sync.Pool`, causing use-after-recycle data races.

## Impact
Potential panics, corrupted translations, or nil pointer dereferences under load. Timing-dependent but possible when connection reuse and `sync.Pool` recycling are active.

## Suggestion
Extract all needed i18n strings from `r` before spawning the goroutine and pass them as parameters.
