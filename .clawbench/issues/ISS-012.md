## Description
`CancelExecution` calls `CancelFunc()` but relies on deferred cleanup in `executeTask` to update DB status and publish events. If the CLI process is unresponsive and the event channel never closes, `executeTask` hangs indefinitely. There is no timeout on `executeTask`.

## Impact
Hung CLI process means a permanent zombie execution. No force-cancel mechanism exists for scheduled tasks.

## Suggestion
Add a timeout context to `executeTask`. Consider a `ForceCancel` path that kills the CLI process directly and ensures DB status is updated even if the channel drain hangs.
