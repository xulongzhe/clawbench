## Description
TOCTOU race condition in path validation: `ValidatePath` resolves symlinks at check-time via `filepath.EvalSymlinks`, but returns the unresolved lexical `absPath` for subsequent file operations. Between validation and operation, a symlink can be replaced to point outside the project root. Additionally, `ResolveExistingPath` accepts `evalBase` parameter but never validates against it.

## Impact
Arbitrary file deletion, overwrite, or creation outside the project root if an attacker can concurrently modify symlinks in the project directory.

## Suggestion
Use the resolved (evaluated) path for file operations instead of the lexical path. Add boundary check in `ResolveExistingPath` that validates against `evalBase`.
