## Description
DeleteSession not atomic — partial soft-delete leaves inconsistent state. The soft-delete updates the session and its messages in separate operations. If the process crashes or an error occurs between them, some messages may be marked deleted while the session is not (or vice versa).

## Impact
Orphaned messages or sessions in an inconsistent state. Soft-deleted sessions may still appear in queries, or partially deleted sessions may lose messages that should be retained for RAG search.

## Suggestion
Wrap the session and message soft-deletes in a single database transaction. Use `BEGIN` / `COMMIT` to ensure atomicity. Roll back on any error during the operation.
