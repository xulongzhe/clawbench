## Description
`httpDo` and `httpDoWithProject` (helpers.go:80-84) create HTTP requests with no timeout. The `http.Client` used has no `Timeout` field set, meaning requests will hang indefinitely if the server is unresponsive.

## Impact
CLI commands that make HTTP requests (e.g., task operations, RAG queries) will hang forever if the ClawBench server is unresponsive. The user has no way to cancel the hung command other than killing the process, and the CLI provides no feedback about the hang.

## Suggestion
Set a reasonable timeout on the `http.Client` (e.g., 30 seconds for normal operations, longer for file uploads). Make the timeout configurable via a command-line flag or environment variable. Also consider adding a context with timeout to individual requests for finer-grained control.
