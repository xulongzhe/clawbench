## Description
`AsyncRefreshModelCache` updates in-memory agents without synchronization, creating data race with concurrent HTTP handlers reading `agent.Models`. `agent.Models = models` directly mutates a field read concurrently by HTTP handlers. Also, `MergeDiscoveredData` soft-removes agents but `Agents` map and `AgentList` slice are not updated atomically — concurrent readers may see inconsistent state.

## Impact
Go data race: can cause nil pointer dereferences or stale/inconsistent model lists under high concurrency. Concurrent readers between `delete(Agents, agent.ID)` and `AgentList = keep` may see an agent in `AgentList` but not in `Agents`.

## Suggestion
1. Protect `agent.Models` mutation with `sync.RWMutex` or `atomic.Value`.
2. Protect both `Agents` and `AgentList` mutations under a write lock, and ensure concurrent readers acquire the read lock.
