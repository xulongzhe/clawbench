## Description
`CheckContinueSession` (continue_conversation.go:29) has no project ownership validation. The auto-restore logic can restore a soft-deleted session from a different project, causing cross-project data contamination.

## Impact
When a user requests a continue conversation for a task execution, the system may auto-restore a soft-deleted session from a different project. This means the continued session contains messages from a different project's context, potentially exposing sensitive information or causing the AI to operate on the wrong project.

## Suggestion
Add a project path check in `CheckContinueSession` before auto-restoring a session. Verify that the session's project path matches the current project. If they don't match, do not auto-restore and instead create a fresh continued session.
