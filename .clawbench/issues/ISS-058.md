## Description
Cancel race — CancelSession and a goroutine's deferred UnregisterSessionStream can execute concurrently. If the channel is closed by CancelSession before UnregisterSessionStream finishes draining, reads from the closed channel cause a panic or zero-value data.

## Impact
Panic/crash of the server process, or silent data corruption in the SSE stream to the client.

## Suggestion
Use a sync primitive (e.g., `sync.Mutex` or `sync.Once`) to coordinate between cancel and unregister. Ensure the channel is only closed by one party, and that all readers check a "closed" flag before reading. Consider using a `context.Done` pattern instead of closing the data channel directly.
