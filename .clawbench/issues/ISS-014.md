## Description
Type mismatch: `navigateToTaskSettings` expects `number` but receives `string` in TaskTab.vue. `onFormSaved` declares parameter as `string` but the backend returns `task.id` as an int64 JSON number. When `navigateToTaskSettings` sets `selectedTaskId.value = taskId` with a string, `selectedTaskData` computed does `t.id === selectedTaskId.value` — strict equality `number === string` always `false`.

## Impact
The detail/settings page shows no data after creating or selecting a task because the ID comparison always fails.

## Suggestion
Change `onFormSaved(newTaskId: string)` to `onFormSaved(newTaskId: number)` and `onTaskSelect(taskId: string)` to `onTaskSelect(taskId: number)`. Add TypeScript emit types to `TaskFormPage`.
