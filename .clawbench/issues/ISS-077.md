## Description
No project-ownership check on ServeChatHistory — any authenticated user can query chat history for any project, not just the one they are operating in. The handler does not verify that the requested session belongs to the current project context.

## Impact
Cross-project data leak: a user in one project can read chat messages from another project, potentially exposing sensitive information, prompts, and AI responses.

## Suggestion
Add a project-ownership check in ServeChatHistory: verify that the requested session's project matches the authenticated project context (e.g., from the `clawbench_project` cookie or middleware). Return 403 Forbidden if there is a mismatch.
