## Description
onerror + resetStreamTimeout race — when EventSource fires an error, the onerror handler initiates teardown, but the stream timeout callback can fire during this teardown window. This causes the system to oscillate between SSE and polling modes, creating a reconnection loop.

## Impact
Users experience intermittent stream disconnections and reconnections. Chat messages may be delayed or duplicated during the oscillation. The polling fallback activates unnecessarily.

## Suggestion
Use a state machine or flag to prevent the timeout callback from firing during teardown. Clear the timeout in the onerror handler before initiating reconnection logic. Add a "reconnecting" state that suppresses timeout-triggered mode switches.
