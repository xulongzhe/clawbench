## Description
`resume_split` SSE event is forwarded to the frontend by the backend but never handled on the frontend. After `resume_split`, the backend finalizes the current DB message and creates a new streaming placeholder, but the frontend continues appending to the OLD `streamingMsg`. The user sees incorrect coalescing (new text appended to pre-resume message instead of new message bubble) until `done` triggers `onLoadHistory()`.

## Impact
UI shows incorrect message coalescing during AutoResume sessions. New text appears in the wrong message bubble until session completes.

## Suggestion
Add a `resume_split` event listener that mirrors the backend logic: finalize current streaming message, then push a new empty assistant message with `streaming: true` and re-assign `streamingMsg`.
