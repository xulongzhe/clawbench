## Description
Path traversal in ServeIndex — user-controlled URL path concatenated into filesystem path without sanitization. An attacker can craft a request with `../` sequences to read arbitrary files on the server.

## Impact
Arbitrary file read from the server filesystem, potentially exposing secrets, configuration, and source code.

## Suggestion
Sanitize the URL path before joining it with the static directory. Reject requests containing `..` path segments and ensure the resolved path stays within the intended root directory using `filepath.Rel` or a prefix check after `filepath.Clean`.
