## Description

`*http.Request` is captured by closure and passed into goroutines that outlive the HTTP handler. The localizer helper `loc(r)` is called inside these goroutines at chat.go:361, 411, and 422, which accesses the request object after the handler has returned.

## Impact

Violates Go's HTTP contract. If Go's server recycles Request objects, translated strings become garbage. Could panic if `loc(r)` ever accesses Body/Context. This is a data race that can produce corrupted UI text or server panics under load.

## Suggestion

Extract the localizer before spawning the goroutine; pass a locale string or localizer object instead of the raw `*http.Request`. For example, call `loc(r)` once before `go func(...)` and pass the resulting translated strings or locale identifier into the goroutine.
