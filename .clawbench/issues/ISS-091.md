## Description
useSessionIdentity action callbacks are nullable with silent no-ops. When an action callback (e.g., `onModelChange`, `onThinkingEffortChange`) is not provided, the call is silently skipped. There is no log, warning, or error to indicate the action was not performed.

## Impact
Configuration changes (model, thinking effort) are silently lost when callbacks are not wired up. The UI appears to accept the change, but the underlying session state is not updated. Hard to debug because there is no error feedback.

## Suggestion
Make action callbacks required (non-nullable), or provide sensible default implementations. If nullable is intentional, log a warning when the callback is missing and an action is triggered. Consider using an event emitter pattern instead of nullable callbacks.
