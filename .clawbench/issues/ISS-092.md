## Description
Double-close panic in Indexer.Stop()/CleanupWorker.Stop() — there is a race between mutex release and channel close. If Stop() is called concurrently, one goroutine may close the channel while another is still holding the mutex or about to close the same channel, causing a panic on double close.

## Impact
Server panic when RAG indexing is stopped concurrently (e.g., during shutdown or reconfiguration). This can crash the entire process.

## Suggestion
Use `sync.Once` to guard the channel close in Stop(). Ensure the channel is closed exactly once regardless of how many times Stop() is called. Check the channel state before closing, or use an atomic flag to track whether Stop() has already been called.
