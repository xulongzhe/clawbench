## Description
Codex resume path never checks `scanner.Err()` — after the scanner loop exits, the error is silently ignored. Partial output due to a read error (e.g., process killed, pipe broken) is accepted as complete.

## Impact
Truncated AI responses are silently accepted. The user receives incomplete output with no indication that data was lost. This is especially dangerous for code generation where partial output could be syntactically invalid.

## Suggestion
Check `scanner.Err()` after the scan loop. If it is non-nil, emit an error event and log the failure. Do not treat truncated output as a successful completion.
