## Description
VeCLI ExecuteStream silently drops events on non-blocking send. When the output channel is full, events are discarded without logging or buffering. This uses a non-blocking send pattern (`select { case ch <- event: default: }`) that silently drops data.

## Impact
Chat events (text, tool calls, thinking blocks) are lost without any indication to the user. The AI response appears incomplete or garbled. No error is logged, making the issue hard to diagnose.

## Suggestion
Either use a buffered channel large enough to handle burst events, or implement backpressure by blocking (with a timeout) instead of dropping. At minimum, log a warning when events are dropped so the issue is observable.
