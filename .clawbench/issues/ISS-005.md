## Description
`CancelSession` sends "cancelled" to streamCh while AI goroutine may still write to it. There is a race window where: (1) CancelSession writes "cancelled" to the channel, (2) the AI goroutine's defer fires and closes the channel via `UnregisterSessionStream`, (3) the drain loop tries to write a terminal event to the now-closed channel, causing a panic on send to closed channel. Sending to a closed channel always panics in Go, even in a select with default.

## Impact
Server panic on concurrent cancel/session-completion race condition. This is a real crash risk in production.

## Suggestion
Change `sendFinalEvent`/`sendEvent` to use `service.SendSessionEvent` (which does `sessionStreams.Load` + channel send) instead of writing directly to the channel variable. Alternatively, ensure `UnregisterSessionStream` is only called after the AIChat goroutine has fully exited.
