# ClawBench：把 AI 编程带到手机上

![Hero](../.clawbench/generated/img_promo_hero_001.jpg)

现在不少人已经习惯了用 Claude Code、CodeBuddy 这类工具写代码。
很多时候不是我们真的在一行一行敲，而是先把思路说清楚，再让 AI 去读文件、改代码、跑命令、做排查。

问题也很现实：这些能力大多都绑定在电脑前。

你在通勤路上、外出吃饭、躺在沙发上，或者临时碰到线上问题时，脑子里明明已经有思路了，却还是得等回到电脑旁边，才能真正动手。

**ClawBench 想解决的，就是这个问题。**

它不是一个“手机上和 AI 聊天”的壳，而是把原本在终端里干活的 AI 编程工具，尽可能完整地搬到手机上。你不需要守在工位前，照样可以看项目、改文件、盯任务进度，必要时直接让 AI 开始处理。

而且这件事不是停留在设想层面。
**ClawBench 这个项目本身，开发过程中就一直在用 ClawBench 自己来推进。** 很多文档修改、代码查看、临时调整、移动端上的排查和接力，都是直接在这个工具里完成的。某种意义上，它不是先做出来再找场景，而是边用边做、边做边验证。

## 一、ClawBench 是什么

ClawBench 是一个开源的移动端 AI 编程工作站。

你可以在手机浏览器里直接连接 Claude Code、CodeBuddy、OpenCode 这类 AI 编程工具，使用它们原本就具备的能力，而不是只停留在“发一句话、回一段文本”的层面。

换句话说，它更像是 **“把终端里的 AI 编程环境搬到手机上”**，而不是重新做一个简化版 App。

![Terminal to Mobile](../.clawbench/generated/img_promo_terminal2mobile_001.jpg)

它主要做了几件事：

- **保留原生能力**：工具调用、深度思考、Skill 工作流、MCP 插件这些能力都能正常使用，不需要额外做一套移动端适配
- **把文件操作补齐**：浏览、编辑、搜索、上传都能在手机上完成，支持代码高亮和 Git Diff，对项目状态有基本掌控力
- **支持多智能体协作**：不同角色可以分工处理不同类型的任务，也可以通过配置切换不同后端
- **同时兼容多种 AI CLI**：不是只绑定单一工具，CodeBuddy、Claude Code、OpenCode 都能接进来
- **开源且免费**：MIT 协议，代码公开，能自己部署，也方便按需改造

## 二、它和普通手机 AI 工具有啥不一样

现在手机上能用的 AI 产品很多，但大多数更偏“聊天助手”。

你可以问问题、让它写段文案、总结一段内容，但一旦事情进入真正的工程阶段——比如读项目文件、改配置、查日志、执行命令、连续处理多步任务——很多工具就接不上了。

ClawBench 的重点恰恰在这里：

**它不是只负责回答，而是尽量让 AI 继续“干活”。**

![Features](../.clawbench/generated/img_promo_features_001.jpg)

具体来说，手机端里能做的事情包括：

**文件浏览**：支持目录递归浏览、搜索、过滤、排序，也能做重命名、复制、移动、删除这些常用操作。

**代码查看与编辑**：有语法高亮、行号、基础编辑能力，适合在手机上快速改几行代码、看配置、补一段内容。

**Markdown 阅读体验**：渲染模式和源码模式可以切换，目录、LaTeX、Mermaid 这些文档里常见的内容也能正常显示。

**AI 会话与工具调用**：支持流式输出，可以看到 AI 的处理过程、工具调用和返回结果，不只是看一个最终答案。

**媒体与附件预览**：图片、PDF、音频、视频都能直接看，不用来回切其他 App。

**Git 变更查看**：能看提交记录，也能看文件改动和 Diff，适合快速确认这次到底改了什么。

它并不想替代桌面开发环境，但在很多“需要随时接上工作流”的场景里，已经够用了。

## 四、架构其实不复杂

![Architecture](../.clawbench/generated/img_promo_arch_001.jpg)

ClawBench 的思路很直接：

- **手机浏览器 / PWA** 负责界面，基于 Vue 3 + TypeScript，重点是把移动端交互做好
- **Go 后端** 提供 HTTP API 和 SSE 流式通信，负责把前端和各类 AI CLI 连接起来
- **SQLite** 用来保存会话和配置，部署简单，也足够轻量
- **AI CLI**（CodeBuddy / Claude Code / OpenCode）继续作为真正干活的后端能力来源

所以它本质上不是把 AI 能力重新“包装”一遍，而是保留原来的工作方式，只是把操作入口换成了手机。

你可以把它理解成：
**AI 编程智能体跑在服务器或电脑上，手机只是一个更方便随身带着的操控端。**

## 五、哪些场景会比较有用

![Scene](../.clawbench/generated/img_promo_scene_001.jpg)

ClawBench 比较适合下面这些场景：

- **通勤或碎片时间**：看一眼 AI 当前跑到哪一步，有需要就补一句指令，不用专门等到坐回电脑前
- **线上临时处理**：碰到告警、报错、异常日志时，可以先在手机上看情况，让 AI 帮忙做初步定位
- **灵感来得快的时候**：想到一个重构思路、文案修改、脚本调整，可以先开个会话把事推进起来
- **快速 Review 改动**：看看最近提交了什么、Diff 里改了哪些地方，适合做快速确认
- **远程盯进度**：AI 在后台继续处理任务时，手机上能随时查看状态，不容易“断线”
- **项目自己就这么在用**：ClawBench 本身的开发过程中，就有不少内容是在 ClawBench 里完成的，这不是一个只拿来演示的概念产品

它最适合的不是重度开发，而是 **把原本必须回到电脑前才能做的事情，提前一部分在手机上接住。**

## 六、怎么安装

### 下载

可以直接从 [GitHub Releases](https://github.com/xulongzhe/clawbench/releases) 下载对应平台的预编译二进制文件。

```bash
# 1. 下载并解压
wget https://github.com/xulongzhe/clawbench/releases/latest/download/clawbench-linux-amd64.zip
unzip clawbench-linux-amd64.zip

# 2. 配置文件
cd clawbench
cp config.example.yaml config.yaml
# 编辑 config.yaml，至少配置 watch_dir 和 password

# 3. 启动服务
./clawbench-linux-amd64
```

### 手机上访问

服务启动后，在手机浏览器里打开：

`http://你的IP:20000`

就可以直接进入界面。

> 建议顺手“添加到主屏幕”，PWA 模式下用起来会更像一个原生应用。

### 使用前提

如果你想在手机上调用对应的 AI 编程工具，需要先在电脑或服务器上装好相关 CLI：

- **CodeBuddy**：`npm i -g @anthropic-ai/codebuddy`
- **Claude Code**：`npm i -g @anthropic-ai/claude-code`
- **OpenCode**：`npm i -g opencode`

## 七、为什么会做这个项目

这个项目最开始没有什么宏大的目标，原因其实很简单：**自己真的会用到。**

AI 编程工具已经很好用了，但使用场景始终偏“坐在电脑前”。
而很多真实场景并不是这样——你可能在路上、在外面、在等人，或者只是暂时不想打开电脑，但又确实想把手头的事情往前推一点。

所以 ClawBench 从一开始就不是“先想一个产品概念，再去拼功能”的路子，
而是 **我自己先拿它来干活，再一边使用、一边把缺的能力补上。**

也就是说，ClawBench 不只是服务别人的工具，**它本身就是用 ClawBench 这套工作流不断开发出来的。** 这点对我来说很重要，因为它能不能真正在手机上接住开发流程，不靠讲，靠自己每天用。

我想做的，不是一个移动端聊天壳，而是一个尽量不阉割能力的入口。

能看文件，能改内容，能调智能体，能继续接住终端里的那套工作流。
这样手机就不只是“看通知的地方”，而是真的能在一些关键时刻把工作接起来。

所以 ClawBench 的核心想法一直很明确：

**不是把 AI 编程变成手机聊天，而是把原本的 AI 编程能力延伸到手机上。**
