## 1. 概述

### 1.1 功能简介

内置终端模块为用户提供在应用内直接启动一个真实 bash shell 的能力。用户可以在不离开当前项目上下文的情况下执行命令行操作，并支持常用命令管理、启动命令配置、命令历史记录等增强功能。

### 1.2 设计目标

- **高效便捷**：提供与系统终端一致的完整功能体验，同时增强常用操作
- **跨平台支持**：支持桌面端和移动端（Android / iOS WebView）
- **个性化配置**：支持项目级和全局级配置，满足不同项目需求
- **安全隔离**：终端初始工作目录被限制在项目根目录以内
- **智能辅助**：常用命令快速访问、命令历史追踪

### 1.3 用户故事

| 用户角色 | 场景 | 期望 |
|---------|------|------|
| 前端开发者 | 需要频繁运行 `npm run dev` | 通过悬浮按钮快速启动，无需重复输入 |
| 后端开发者 | 项目需要切换 Node.js 版本 | 在项目配置中设置启动命令 `nvm use 18` |
| 全栈开发者 | 需要跨多个项目工作 | 每个项目保持独立的配置和历史记录 |
| 移动端用户 | 需要查看大量日志输出 | 切换滚动模式，方便浏览历史输出 |

## 4. 技术架构

### 4.1 技术选型

| 层级 | 技术 | 说明 |
|---|---|---|
| 后端 | Go + `creack/pty` | 启动真实 PTY 进程（bash shell） |
| 数据库 | SQLite | 持久化配置和命令历史 |
| 传输 | WebSocket (`/api/terminal/ws`) | 双向实时通信 |
| 前端渲染 | xterm.js 5.x + FitAddon | 终端模拟器，通过 npm 包引入 |
| 前端框架 | Vue 3 (Composition API) | 组件化实现 |

### 4.2 数据模型

#### 4.2.1 终端配置 (TerminalConfig)

```go
type TerminalConfig struct {
    ID              string            `json:"id" db:"id"`                    // uuid
    Scope           string            `json:"scope" db:"scope"`              // "global" | "project"
    ProjectID       *string           `json:"projectId,omitempty" db:"project_id"`
    StartupCommands []string          `json:"startupCommands" db:"startup_commands"`  // JSON array
    Shell           string            `json:"shell" db:"shell"`             // 默认 "/bin/bash"
    Env             map[string]string `json:"env" db:"env"`                  // JSON object
    FontSize        int               `json:"fontSize" db:"font_size"`     // 默认 13
    CreatedAt       time.Time         `json:"createdAt" db:"created_at"`
    UpdatedAt       time.Time         `json:"updatedAt" db:"updated_at"`
}
```

#### 4.2.2 常用命令 (Snippet)

```go
type Snippet struct {
    ID        string    `json:"id" db:"id"`            // uuid
    Name      string    `json:"name" db:"name"`       // 显示名称
    Command   string    `json:"command" db:"command"`  // 实际命令
    Scope     string    `json:"scope" db:"scope"`      // "global" | "project"
    ProjectID *string   `json:"projectId,omitempty" db:"project_id"`
    Order     int       `json:"order" db:"order"`     // 排序权重
    Tags      []string  `json:"tags" db:"tags"`        // JSON array
    UseCount  int       `json:"useCount" db:"use_count"`
    LastUsed  *time.Time `json:"lastUsed,omitempty" db:"last_used"`
    CreatedAt time.Time `json:"createdAt" db:"created_at"`
    UpdatedAt time.Time `json:"updatedAt" db:"updated_at"`
}
```

#### 4.2.3 命令历史 (CommandHistory)
