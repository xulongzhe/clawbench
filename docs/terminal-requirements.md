# 终端功能需求文档

> 项目: Markdown Browser
> 版本: v1.0
> 日期: 2026-04-08

---

## 1. 概述

### 1.1 功能简介

内置终端模块为用户提供在应用内直接启动一个真实 bash shell 的能力。用户可以在不离开当前项目上下文的情况下执行命令行操作，终端自动定位于当前选中文件所在的目录。

### 1.2 设计目标

- 提供与系统终端一致的完整功能体验
- 支持桌面端和移动端（Android / iOS WebView）
- 主题随应用明暗模式实时切换
- 安全隔离：终端工作目录被限制在项目根目录以内

---

## 2. 技术架构

### 2.1 技术选型

| 层级 | 技术 | 说明 |
|---|---|---|
| 后端 | Go + `creack/pty` | 启动真实 PTY 进程（bash shell） |
| 传输 | WebSocket (`/api/terminal/ws`) | 双向实时通信 |
| 前端渲染 | xterm.js 5.x + FitAddon | 终端模拟器，通过 npm 包引入 |
| 前端框架 | Vue 3 (Composition API) | 组件化实现 |

### 2.2 组件结构

```
App.vue
└── TerminalPanel.vue          # 终端主组件
    ├── BottomSheet            # 全屏模态容器（独立组件）
    │   └── #header slot       # 标题栏（标题 + 目录路径，内联实现）
    ├── terminal-touch-layer   # 触摸手势层
    │   ├── hidden-input       # 隐藏 input（捕获移动端键盘输入）
    │   └── terminal-body      # xterm.js 渲染容器
    ├── vkey-toast             # 手势反馈提示
    └── vkey-bar               # 移动端虚拟按键栏（内联实现）
```

## 3. 后端功能需求

### 3.1 PTY 会话管理

| 需求编号 | 描述 | 验收标准 |
|---|---|---|
| BE-001 | 使用 `creack/pty` 创建 Pseudo-Terminal | 启动后可在 xterm.js 中看到 bash 提示符 |
| BE-002 | PTY 内运行真实 `/bin/bash` | 支持 bash 所有内置命令和特性 |
| BE-003 | 环境变量 `TERM=xterm-256color` | vim、less、tmux 等工具正常工作，256 色可用 |
| BE-004 | PTY 大小动态可调 | 通过 WebSocket resize 消息实时修改 cols/rows |
| BE-005 | PTY 进程在 WebSocket 断开时自动销毁 | 避免僵尸进程 |
| BE-006 | PTY 错误（读写失败）时 goroutine 干净退出 | 无 panic，不残留 goroutine |

### 3.2 WebSocket 传输

| 需求编号 | 描述 | 验收标准 |
|---|---|---|
| BE-007 | 路由 `/api/terminal/ws` | WebSocket 连接成功建立 |
| BE-008 | 可选查询参数 `?dir=<path>` 限制工作目录 | 传入 `/api/terminal/ws?dir=src/utils` 时，bash `pwd` 返回 src/utils |
| BE-009 | 初始工作目录安全校验 | 传入非法路径（路径穿越、不存在目录）时返回 403/404；bash 仍可 `cd` 到任意目录（仅限制启动位置） |
| BE-010 | 二进制帧传输 PTY 输出 | xterm.js 接收到完整的终端回显，无截断 |
| BE-011 | JSON 消息解析: `type=data` | 前端发送的按键数据正确写入 PTY stdin |
| BE-012 | JSON 消息解析: `type=resize` | PTY cols/rows 与前端一致 |
| BE-013 | JSON 消息解析: `type=ping` | 前端发送心跳消息，服务端暂不处理（可忽略） |

### 3.3 心跳与连接管理

| 需求编号 | 描述 | 验收标准 |
|---|---|---|
| BE-014 | 服务端每 15 秒发送 WebSocket 原生 ping 帧 | 客户端断网后服务端能检测到连接失效并清理资源 |
| BE-015 | 客户端消失后服务端自动清理资源 | PTY 进程被 kill，无 goroutine 泄漏 |

> **说明**：心跳机制分为两套独立实现：
> - **服务端 → 客户端**：使用 WebSocket 原生 ping/pong 帧（非 JSON 消息），由 `terminal.go:69-77` 实现
> - **客户端 → 服务端**：前端发送 JSON `{type: "ping"}` 消息，服务端当前未处理，仅用于保持 WebSocket 连接活跃

## 4. 前端终端渲染需求

### 4.1 终端模拟器配置

| 需求编号 | 描述 | 验收标准 |
|---|---|---|
| FE-001 | 使用 xterm.js 5.x 版本 | `window.Terminal` 构造成功 |
| FE-002 | 加载 FitAddon 并调用 fit() | 终端填满 BottomSheet 容器，无溢出或留白 |
| FE-003 | 字体设置为 `Menlo, Monaco, "Courier New", monospace`，13px | 字体与设置一致 |
| FE-004 | `cursorBlink: true` | 光标闪烁 |
| FE-005 | `macOptionIsMeta: true` | macOS Option 键作为 Meta 键使用 |
| FE-006 | `allowProposedApi: true` | FitAddon 正常工作（FitAddon 依赖 proposed API） |
| FE-007 | `scrollback: 10000` | 可回滚查看最近 10,000 行历史输出 |
| FE-008 | 使用 npm 包 `@xterm/xterm` + `@xterm/addon-fit` | 通过 `import` 引入，非 CDN 加载 |

### 4.2 主题支持

| 需求编号 | 描述 | 验收标准 |
|---|---|---|
| FE-009 | Light 主题: 背景 `#ffffff`，前景 `#212529` | 明模式时终端配色为浅色 |
| FE-010 | Dark 主题: 背景 `#0d1117`，前景 `#e6edf3` | 暗模式时终端配色为深色（GitHub Dark） |
| FE-011 | 完整 16 色调色板（标准色 + 亮色） | `ls --color=auto` 等命令输出颜色正确 |
| FE-012 | 主题随应用明暗模式实时切换 | 切换主题时 xterm.js 主题同步更新，无闪烁 |
| FE-013 | 选择文本高亮色区分 | 明/暗主题各有对应的选择背景色 |

### 4.3 动态尺寸调整

| 需求编号 | 描述 | 验收标准 |
|---|---|---|
| FE-014 | 容器高度 = 可视视口高度 - header 高度 - 虚拟按键栏高度 - 软键盘高度 | 终端面板不超出屏幕，不被虚拟按键/软键盘遮挡 |
| FE-015 | `visualViewport` API 作为主要 resize 信号 | 移动端键盘打开/关闭时终端自动 fit 并发送 resize |
| FE-016 | `ResizeObserver` 作为视觉Viewport不支持时的降级方案 | 桌面浏览器正常工作 |
| FE-017 | WebSocket 连接建立后首次 fit 并发送 resize | PTY 在初始化时即有正确的 cols/rows |
| FE-018 | 每次视口变化时再次 fit 并发送 resize | 旋转屏幕、键盘开合时终端尺寸正确 |

## 8. 非功能性需求

### 8.1 性能

| 需求编号 | 描述 |
|---|---|
| NF-001 | PTY 输出到 xterm.js 渲染延迟 < 50ms（本地网络） |
| NF-002 | 终端 resize 响应时间 < 100ms |
| NF-003 | xterm.js 内存占用合理：10,000 行滚动缓冲（每行约 80 字符 ≈ 800KB） |

### 8.2 安全

| 需求编号 | 描述 |
|---|---|
| NF-004 | 终端**初始工作目录**被限制在项目根目录以内（用户仍可在 bash 中 `cd` 到任意位置） |
| NF-005 | 路径穿越攻击（`../`）在**初始目录参数**中被服务端检测并拒绝 |
| NF-006 | 每次 WebSocket 连接独立 PTY 进程，会话间无数据泄露 |

### 8.3 兼容性

| 需求编号 | 描述 |
|---|---|
| NF-007 | 支持主流桌面浏览器: Chrome 90+、Firefox 90+、Safari 15+、Edge 90+ |
| NF-008 | 支持移动端 WebView: Android Chrome WebView、iOS Safari WebView |
| NF-009 | 优雅降级：无 `visualViewport` API 时使用 `ResizeObserver` 替代 |
